######### Commit Type 종류 #########
# Feat : 기능 추가 작업에 대한 커밋
# Update: 기능 수정 작업에 대한 커밋
# Fix : 버그 코드 수정에 대한 커밋(기능상 오류가 있어야하며 기획에 맞지 않거나 에러에 대한 동작에 대한 코드 수정)
# Build : 빌드 관련 작업에 대한 커밋(runtime에 필요한 코드 이외의 모든 코드)
# Docs : 문서 작업에 대한 커밋(ex. readme.txt 수정)
# Style : UI 스타일 코드 작업에 대한 커밋
# Refactor : 코드 리팩토링 작업에 대한 커밋(버그가 없는 상황에서 개선 목적)
# Test : 테스트 코드 작업에 대한 커밋
# Delete : 쓸모없는 코드 및 파일 삭제에 대한 커밋(ex. 사용하지 않는 패키지, 메서드 및 파일 삭제)
